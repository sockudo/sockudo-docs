## Backend: Laravel Broadcasting (`pusher`)

In Laravel, keep broadcaster as `pusher` and point host/port to Sockudo.

```php [config/broadcasting.php]
'connections' => [
    'pusher' => [
        'driver' => 'pusher',
        'key' => env('PUSHER_APP_KEY'),
        'secret' => env('PUSHER_APP_SECRET'),
        'app_id' => env('PUSHER_APP_ID'),
        'options' => [
            'host' => env('PUSHER_HOST', '127.0.0.1'),
            'port' => (int) env('PUSHER_PORT', 6001),
            'scheme' => env('PUSHER_SCHEME', 'http'),
            'useTLS' => env('PUSHER_USE_TLS', false),
            'cluster' => env('PUSHER_APP_CLUSTER', 'mt1'),
        ],
    ],
]
```

`.env` example:

```bash
BROADCAST_DRIVER=pusher
PUSHER_APP_ID=app-id
PUSHER_APP_KEY=app-key
PUSHER_APP_SECRET=app-secret
PUSHER_HOST=127.0.0.1
PUSHER_PORT=6001
PUSHER_SCHEME=http
PUSHER_USE_TLS=false
PUSHER_APP_CLUSTER=mt1
```

## Frontend: Echo Setup

```ts [bootstrap.ts]
import Echo from "laravel-echo";
import Pusher from "pusher-js";

window.Pusher = Pusher;

window.Echo = new Echo({
  broadcaster: "pusher",
  key: "app-key",
  wsHost: "127.0.0.1",
  wsPort: 6001,
  wssPort: 443,
  forceTLS: false,
  enabledTransports: ["ws", "wss"],
  cluster: "mt1",
  authEndpoint: "/broadcasting/auth",
});
```

## Works Out of the Box

- Public/private/presence channels.
- Laravel auth endpoint flow.
- Broadcast events and whisper/client events behavior.

## Limitations with Echo + pusher-js

Echo does not expose Sockudo-specific APIs for:

- channel subscription filters.
- delta compression negotiation and decoding.

Use `@sockudo/client` for those advanced features (usually via custom app-level client code, outside default Echo abstractions).
